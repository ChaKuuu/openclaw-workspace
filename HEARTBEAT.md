# HEARTBEAT.md

> 每个 Heartbeat 周期检查的内容（30分钟一次）

---

## 每次 Heartbeat 检查

**用户相关：**
- [ ] 有没有需要立即回复的重要消息？
- [ ] 用户有没有提到新的偏好/习惯？（发现就更新 USER.md）

**任务相关：**
- [ ] 有没有悬而未决、等待用户回复的事项？
- [ ] 正在执行的任务有没有遇到障碍需要上报？

---

## 轮流检查（不要每次都全量检查）

### 第1-3个心跳：收件箱 + 消息
- 检查飞书/其他平台的新消息
- 检查有没有需要处理的通知

### 第4个心跳：工程控制论进度
- 当前学到了哪个概念？
- 有没有值得写进笔记的想法？

### 第5个心跳：用户建模
- 这次对话用户透露了什么新信息？
- USER.md 需要更新吗？

### 第6个心跳：自我改进
- 今天有没有新的 corrections 要记录？
- 有没有发现技能可以改进的地方？

---

## 触发 Hermes 风格 Nudge 的条件

当以下任一条件满足时，在下一次 heartbeat 主动发起提醒：

1. **连续 3 个 heartbeat 没有记录任何新发现** → 提醒"该复盘了"
2. **检测到重复错误（第2次出现）** → 提醒"记录到 corrections"
3. **用户提供了新的项目/任务信息** → 提醒"更新 USER.md"
4. **能量函数 V(x) 连续上升** → 提醒"Bang-Bang 急停"
5. **有技能使用后效果不好** → 提醒"技能自评"

---

## 简化原则

- 每次 heartbeat 最多做 **2 件有价值的事**
- 不重复做已经做过的事（除非状态变了）
- 发现新东西 → 立即记录，不要依赖记忆
- 超过 5 分钟还没结论 → 跳过，下次再说
