# Self-Evolution 技能自改进激活手册

> 把 self-evolution 从"框架文档"变成"每次任务后自动触发的闭环"

---

## 触发条件：什么时候该自评？

**必须触发（任务结束）：**
- 完成了一个复杂任务（比如写了一个完整方案、解决了一个 bug）
- 用户对某个交付物明确表示满意或不满意
- 经历了一次失败/挫折/走弯路

**建议触发（每周定时）：**
- 周三 20:00 周中自检时（结合工程控制论 PID 自诊断）
- 每晚 22:00 Nudge 提醒时

**不要触发（除非有实质内容）：**
- 简单的问答（"今天天气怎么样"）
- 纯闲聊没有产生有价值信息

## 任务结束自评模板（复制使用）

```
【任务自评】- {时间}

任务描述：{简单描述完成了什么}

✅ 做得好：
- {哪些做法有效？}
- {哪些决策正确？}

⚠️ 可以改进：
- {哪里可以更好？}
- {哪个环节浪费了时间？}

📝 值得记录：
- {这个经验以后还用得上吗？}
- {有没有值得加入技能的重复模式？}

🔧 技能更新：
- [ ] 需要更新现有技能文档
- [ ] 需要创建新技能
- [ ] 只需要记录到 self-improving

❌ 错误记录（如有）：
{描述错误} → {根本原因} → {以后怎么避免}
→ 追加到 ~/self-improving/corrections.md
```

---

## 技能文档更新标准

**什么情况值得更新技能文档：**
- 同样的任务你用了更好的方法完成
- 发现了一个之前没写到的子任务/边界情况
- 用户提供了更清晰的期望/标准
- 某个技能用起来效果不好，需要调整

**更新步骤（Safe Self-Modification）：**
1. 备份原文件
2. 写入改进内容
3. 简单验证（文件可读）
4. 记录到 evolution log

**技能文档位置约定：**
- 通用技能 → `~/.openclaw/workspace/skills/<skill-name>/`
- 项目特定 → 项目目录下的 README.md
- 工程控制论 → `notes/工程控制论/`

## 与现有 Cron 任务的衔接

```
每晚 22:00 Nudge
  ↓
触发 self-evolution 自评
  ↓
检查：有没有今天值得记录的新发现？
  ↓
有 → 按模板记录，写入对应文件
没有 → 跳过，不做无用功
```

```
每周三 20:00 PID 自检
  ↓
结合 self-evolution 自评
  ↓
P/I/D 哪种模式主导？
  ↓
如果 D 模式过强（想太多）→ 提醒：减少分析瘫痪，立即行动
```

## 快速开始

**从今天开始，每次任务后：**
1. 在心里过一遍上面的自评模板（30秒）
2. 如果有值得记录的 → 写 3-5 句话到对应文件
3. 复杂的改进 → 创建正式的自评记录

**不是每次都要写完整报告** —— 哪怕只记一句话也好过什么都不留。
