## 一、任务结束后的自改进闭环（Hermes 风格）

> 每个任务/对话结束后自动触发，不需要等待定时

### 触发时机
- 每次完成一个复杂任务后
- 每周定时也会触发（通过 Nudge）

### 执行步骤

```
【任务结束自评 → Hermes Skill Self-Improvement Loop】

Step 1: 回顾任务
  - 任务目标是什么？完成了吗？
  - 遇到了什么阻碍？怎么解决的？
  - 有没有重复出现的模式？

Step 2: 判断是否值得创建/改进技能
  - 这个经验以后还会遇到吗？→ 值得写进技能文档
  - 这次用的方法比技能里写的更好吗？→ 值得更新技能
  - 有没有新的子任务类型出现？→ 值得创建新技能

Step 3: 如需要，更新技能文档
  - 用 safe_self_modification 协议：
    Backup → Modify → Test → Rollback if failed → Log
  - 写入 notes/hermes-agent/skills-进化日志.md

Step 4: 更新用户模型（Hermes Honcho 风格）
  - 这次对话用户透露了什么偏好？
  - 用户的决策模式是什么？
  - 更新 notes/USER.md

Step 5: 更新自我改进日志
  - 记录这次学到了什么
  - 记录下次要避免的错误
  - 追加到 ~/self-improving/corrections.md
```

## 三、周反馈日执行流程（工程控制论 + Hermes）

> 每周一 09:00 触发（已在 cron 注册）

### Personal ACS 算法执行

```
【周反馈日 → Personal ACS 闭环】

Phase 1: 预测阶段（Predict）
  1. 读取上周预测 vs 实际结果
  2. 计算偏差 e = 实际 - 预测
  3. 分类偏差来源：
     - 系统性偏差（模型问题）→ 调整参考模型
     - 随机干扰（一次性因素）→ 忽略
     - 反馈延迟导致超调 → 缩短反馈周期
     - 环境突变 → 触发自适应重识别

Phase 2: 状态更新（Update）
  4. 卡尔曼滤波更新：
     x̂(k|k) = x̂(k|k-1) + K(z(k) - h(x̂(k|k-1)))
     - Q（过程噪声）↑ → 相信自己模型少一点
     - R（观测噪声）↑ → 更相信模型预测
  5. 评估 Q/R 比值是否合理

Phase 3: 能量检查（Lyapunov）
  6. 计算 V(x) = 焦虑² + 愧疚² + 低效² + 关系张力²
  7. 检查 V̇(x) 趋势
  8. 如 V̇(x) > 0：触发 Bang-Bang 急停

Phase 4: 策略调整（Control）
  9. PID 自诊断：
     - P 过大 → 振荡 → 减少 Kp
     - I 累积误差太大 → 降低 Ki
     - D 过强 → 响应迟缓 → 减少 Kd
  10. 更新下周控制策略

Phase 5: 文档化（Document）
  11. 写入 notes/工程控制论/每周复盘.md
  12. 更新 notes/hermes-agent/skills-进化日志.md
```

## 四、技能自改进协议（安全自修改）

### 修改权限分级

**✅ 可以自行修改（无需询问）：**
- 技能文档的示例代码和说明文字
- 复盘模板和执行流程
- 笔记文件的内容补充
- `~/self-improving/` 下的学习日志

**❌ 必须先询问再操作：**
- 删除任何文件
- 发送外部消息（邮件/消息/帖子）
- 修改系统级配置
- 改动用户数据
- API key 或凭证相关操作

### Safe Self-Modification 协议

```
对每个技能修改：

1. Backup  → 复制原文件到备份
2. Modify  → 应用修改
3. Test    → 简单验证（文件存在、可读、无语法错误）
4. Rollback if failed → 恢复备份
5. Log     → 记录到 evolution log
```

## 五、用户建模更新流程（Hermes Honcho 风格）

### 更新触发时机
- 每次对话结束
- 每晚 Nudge 检查
- 用户明确提供了新偏好/信息

### 用户建模维度

```
【USER.md 更新检查清单】

每次检查是否需要更新：
□ 工作偏好：喜欢详细回复还是简洁回复？
□ 沟通风格：对技术文档的反应如何？
□ 决策模式：喜欢快速行动还是深思熟虑？
□ 关注领域：最近在关注什么主题？
□ 工作节奏：什么时间段效率最高？
□ 反馈偏好：喜欢直接指出问题还是委婉提醒？
□ 特殊习惯：有什么我应该知道的工作习惯？
```

### 更新原则
- 只记录观察到的行为模式，不做假设
- 用户主动提供的信息优先于推测
- 定期回顾 USER.md，删除过时信息

## 八、到 Hermes 的迁移准备

当 WSL2 就绪后，执行：

```bash
# 1. 安装 Hermes
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash

# 2. 从 OpenClaw 迁移
hermes claw migrate --dry-run    # 先预览
hermes claw migrate             # 执行迁移

# 3. 迁移内容确认：
# ✅ SOUL.md → Hermes persona
# ✅ USER.md → Honcho 用户模型
# ✅ MEMORY.md → Hermes memory
# ✅ Skills → ~/.hermes/skills/openclaw-imports/
# ✅ 飞书配置 → Hermes messaging gateway
```
