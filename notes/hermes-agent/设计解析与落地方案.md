# Hermes Agent 核心设计解析 & OpenClaw 落地方案

> 基于 NousResearch/hermes-agent README 分析

---

## 一、核心架构对比

| 功能模块 | Hermes Agent | OpenClaw（当前） | 差距 |
|---------|-------------|-----------------|------|
| Agent Loop | 观察→规划→执行→反思→记忆 | 基本循环已有 | 缺自动反思闭环 |
| 技能系统 | 使用中自动改进，事后评估 | self-evolution 框架在，但未激活 | 需要激活循环 |
| 记忆系统 | FTS5全文搜索 + Honcho用户建模 | memory_search 语义搜索 | 缺主动用户建模 |
| Nudge机制 | 周期性"提醒自己保存知识" | 无 | **需要实现** |
| 跨平台接入 | Telegram/Discord/Slack/WhatsApp | 飞书/Webchat | OpenClaw 更好 |
| Cron调度 | 自然语言配置 | cron 工具已有 | 差距不大 |
| 子Agent | RPC调用，并行工作流 | sessions_spawn | OpenClaw 已有 |

## 二、最值得学习的三个设计

### 设计1：Nudge 机制（定期提醒自己）
**Hermes 做法：** 周期性弹出 nudge，提醒 agent 反思最近经验、判断是否需要创建/改进技能、保存知识。

**OpenClaw 落地：**
```
每晚 22:00 Nudge 检查：
  1. 今天完成了什么？有没有值得提取成技能的？
  2. 有没有重复犯的错？该记录到 self-improving？
  3. 用户透露了什么新偏好/信息？该更新到 USER.md？
  4. 有没有悬而未决的问题？该留个待办？
```

### 设计2：技能使用中自改进
**Hermes 做法：** 每次技能使用后评估效果，如果效果好→强化该技能模式，如果效果差→生成改进方案。

**OpenClaw 落地：**
```
技能执行后自评（嵌入 self-evolution）：
  - 这个技能用了多久？效果如何？
  - 有没有遇到阻碍？根因是什么？
  - 这次经验可以写进技能文档吗？
  - 下次怎么改进？
```

### 设计3：Honcho 用户建模
**Hermes 做法：** 每次对话后用 LLM 分析，动态更新用户画像（偏好、习惯、弱点）。

**OpenClaw 落地：**
```
每次对话结束后（通过 HEARTBEAT 或 cron）：
  - 这次对话用户透露了什么新信息？
  - 用户的决策模式有没有变化？
  - 该更新 USER.md 吗？
```

## 四、OpenClaw → Hermes 迁移清单

（等 WSL2 就绪 + Hermes 稳定后执行）

```bash
# Hermes 提供官方迁移工具
hermes claw migrate --dry-run    # 预览
hermes claw migrate             # 执行迁移

# 迁移内容：
# ✅ SOUL.md、USER.md、MEMORY.md
# ✅ Skills（→ ~/.hermes/skills/openclaw-imports/）
# ✅ API keys（白名单内的）
# ✅ 飞书配置（如果有）
# ✅ Workspace instructions（AGENTS.md）
```

## 五、当前环境不适用的 Hermes 功能

以下功能在 OpenClaw 环境下暂时不需要追求：

- **多消息平台接入**：OpenClaw 已有飞书，够用
- ** Daytona/Singular/Modal 部署**：需要云基础设施
- **RL Training（Tinker-Atropos）**：训练能力，当前不需要
- **batch trajectory 生成**：研究用途，当前不需要
