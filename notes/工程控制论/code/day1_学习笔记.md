## 1. 核心公式推导

### 传递函数定义
在零初始条件下：
$$G(s) = \frac{Y(s)}{U(s)}$$

### 4种典型环节

| 环节 | 传递函数 | 阶跃响应 | 稳态值 |
|------|---------|---------|-------|
| 比例 | K | y(t) = K | K |
| 惯性 | K/(Ts+1) | y(t) = K(1-e^(-t/T)) | K |
| 积分 | K/s | y(t) = K·t | ∞ |
| 微分(实际) | K·s/(Ts+1) | y(t) = (K/T)·e^(-t/T) | 0 |

---

## 2. 惯性环节时间常数 T 的物理意义

$$y(t) = K \cdot (1 - e^{-t/T})$$

| t | y(t)/K | 百分比 |
|---|--------|-------|
| 1T | 1-e^(-1) | **63.2%** |
| 2T | 1-e^(-2) | 86.5% |
| 3T | 1-e^(-3) | 95.0% |
| 4T | 1-e^(-4) | 98.2% |
| 5T | 1-e^(-5) | 99.3% |

> **T 是系统达到最终值 63.2% 所需的时间。**
> 工程上认为 5T 后系统到达稳态（99.3%）。

---

## 3. Python 仿真验证

```python
from scipy import signal
import numpy as np

# 惯性环节: K/(Ts+1)
K, T = 2.0, 1.0
sys = signal.TransferFunction([K], [T, 1])
t, y = signal.step(sys, T=np.linspace(0, 5, 1000))

# 解析解
y_analytical = K * (1 - np.exp(-t/T))
```

数值解与解析解完全吻合 ✅

## 4. 工程控制论应用

### 传递函数 = 能力的"指纹"

类比到个人能力系统：
- **比例环节 K**：直接输出，如"已有技能直接应用"
- **惯性环节 T**：能力需要时间发酵，如"新语言学习"
- **积分环节**：持续积累，如"每天背单词"
- **微分环节**：趋势预测，如"势头分析"

### 时间常数 T 的实际意义
- 新技术 T=3个月：3个月后才算真正学会（95%）
- 健身 T=6周：6周后体能稳定在某个水平
- 写作 T=1年：写1年后才感觉"能写了"

---

## 5. 关键洞察

> **T 不是"完成时间"，而是"达到63.2%的时间"。**
> 系统永远不会真正"完成"，只会无限接近稳态。

> **积分环节的稳态值 = ∞，这意味着纯粹的积分器是不稳定的。**
> 实践中积分总是和比例、微分配合（PID），单独积分会导致系统失控。
