# Day 3 学习笔记：Routh-Hurwitz 稳定性判据

> 日期：2026-04-14 | 用时：约30分钟

---

## 1. 核心思想

**不需要求极点，就能判断稳定性。**

对于 n 阶系统：
$$a_ns^n + a_{n-1}s^{n-1} + \cdots + a_1s + a_0 = 0$$

稳定条件：**所有极点实部 < 0**（左半平面）

Routh-Hurwitz 提供了**直接看系数判断稳定性**的方法。

---

## 2. Routh 数组构建规则

对于特征方程系数：[a_n, a_{n-1}, ..., a_1, a_0]

**第一行：** a_n, a_{n-2}, a_{n-4}, ...
**第二行：** a_{n-1}, a_{n-3}, a_{n-5}, ...

**后续行：** 用行列式计算
$$b_i = \frac{a_{i+1} \cdot (第一列第二个) - a_1 \cdot (上一行第i+2个)}{a_1}$$

**第一列符号变化次数 = 右半平面极点个数**

---

## 3. 稳定性判断准则

**充要条件：** Routh 数组**第一列**所有元素**同号**（全正或全负）

| 系统 | 第一列 | 判定 |
|------|--------|------|
| s³+2s²+3s+4 | [3, 4, 0.5] 全正 | **稳定** ✅ |
| s³+2s²+3s-4 | [3, -4, -2.5] 有负 | **不稳定** ❌ |
| s⁴+2s³+3s²+4s+5 | [5, 4, -0.5, 6] 有负 | **不稳定** ❌ |

---

## 4. 临界稳定与增益 K

**例子：** G(s) = K / (s³ + 2s² + 3s + K)

求使系统稳定的 K 范围：

```
K < 6.023 → 稳定
K = 6.023 → 临界稳定（持续振荡）
K > 6.023 → 不稳定
```

**工程意义：** 增益过大导致系统振荡甚至失稳。

---

## 5. 数值验证（Python + scipy）

```python
import numpy as np

coeffs = [1, 2, 3, 4]  # s³ + 2s² + 3s + 4
poles = np.roots(coeffs)

# 极点
poles = [-1.65, -0.175+1.55j, -0.175-1.55j]

# 稳定性
all(p.real < 0 for p in poles)  # True → 稳定
```

scipy 验证结果：
- s³+2s²+3s+4 → 3个极点全在左半平面 → **稳定** ✅
- s³+2s²+3s-4 → 有极点实部 > 0 → **不稳定** ❌

---

## 6. 第一列元素的物理含义

**第一列元素 = 劳斯数组的"稳定裕度"**

- 值越大 → 越稳定（稳定裕度大）
- 值接近 0 → 接近临界稳定
- 变为负 → 失稳

**工程应用：** 调整控制器增益后，用 Routh 判据快速检查稳定性，不必每次都求根。

---

## 7. 特殊情况

| 情况 | 处理方法 |
|------|---------|
| 第一列出现 0 | 用小正数 ε 代替 |
| 整行全为 0 | 用**辅助多项式**系数代替 |
| 辅助多项式阶数为奇数 | 系统处于**临界稳定**边界 |
