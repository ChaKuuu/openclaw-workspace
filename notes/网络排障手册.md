## 网络问题诊断流程

```
症状出现
  ↓
Step 1: 确认范围 — 是全局还是特定目标？
  ├─ 全局（GIT/浏览器/API全挂）→ DNS/网关/ISP
  └─ 特定（只有某服务）→ 该服务专项排查
  ↓
Step 2: 分层测试
  ├─ 第1层: ping / TCP连接测试
  ├─ 第2层: DNS解析测试
  ├─ 第3层: TLS/证书测试
  └─ 第4层: 应用层测试（curl -v）
  ↓
Step 3: 对症修复（见下方）
```

## 常见症状 → 快速修复

### 症状：POST 请求超时 / GET 正常

**今晚 GitHub 的典型案例：**
- GET `github.com` → 200 ✅
- GET `api.github.com` → 200 ✅  
- **POST `github.com/login/device/code` → 超时 ❌**

**原因：** 防火墙/代理只屏蔽特定端点的 POST

**诊断命令：**
```powershell
# 测试 GET（成功）
curl https://github.com -I --max-time 5

# 测试 POST（失败）
curl -X POST https://github.com/login/device/code -I --max-time 5
```

**修复方案：**
1. 换用 `--with-token` 替代 `--web` 设备码流程（但仍需 POST）
2. 浏览器手动获取 token，粘贴给我
3. 配置代理允许 POST 到目标域名
4. 使用 GitHub CLI 绕过（`gh auth`）

### 症状：DNS 解析失败

**诊断命令：**
```powershell
# Windows 原生命令
nslookup example.com
nslookup example.com 8.8.8.8    # 指定 DNS 服务器

# Windows Ping（同时测 DNS）
ping example.com
```

**常见原因：**
| 原因 | 修复 |
|------|------|
| DNS 服务器无响应 | 换用 8.8.8.8 或 1.1.1.1 |
| 本地 DNS 缓存 | `ipconfig /flushdns` |
| /etc/hosts 错误覆盖 | 检查 hosts 文件 |
| CDN 解析到错误 IP | 等待 TTL 过期或降低 TTL 后重试 |

### 症状：连接超时（Connection Timeout）

**诊断命令：**
```powershell
# TCP 端口检测（无 HTTP 的端口）
Test-NetConnection example.com -Port 443
Test-NetConnection example.com -Port 22

# 或用 PowerShell
$tcp = New-Object System.Net.Sockets.TcpClient
$tcp.Connect("example.com", 443)
```

**常见原因：**
| 原因 | 修复 |
|------|------|
| 防火墙屏蔽端口 | 开放端口或换端口 |
| 服务未启动 | 启动目标服务 |
| NAT 映射缺失 | 配置端口转发 |
| ICMP 被禁（ping不通） | 用 TCP 端口检测替代 ping |

### 症状：SSL/TLS 证书错误

**诊断命令：**
```powershell
# 检查证书
curl -v https://example.com 2>&1 | Select-String "SSL|TLS|certificate"

# Windows
powershell -Command "[Net.ServicePointManager]::ServerCertificateValidationCallback = {$true}; Invoke-WebRequest https://example.com"
```

**常见原因：**
| 原因 | 修复 |
|------|------|
| 证书链不完整 | 安装中间证书 |
| 证书过期 | 续期证书 |
| SNI 不匹配 | 确认域名正确 |
| 自签名证书 | 信任或忽略（测试环境）|

## HTTP 状态码速查

| 状态码 | 含义 | 常见原因 |
|--------|------|---------|
| 200 | OK | 正常 |
| 301/302 | 重定向 | URL 变化，跟随重定向 |
| 400 | Bad Request | 参数错误、请求格式错误 |
| 401 | Unauthorized | 未认证、Token 过期 |
| 403 | Forbidden | 权限不足、IP 被禁 |
| **429** | Rate Limit | 请求过快，需要等待 |
| 500 | Server Error | 远程服务器故障 |
| 503 | Service Unavailable | 服务过载或宕机 |
| **504** | Gateway Timeout | 代理/网关超时 |

## 今晚 GitHub 排查经验

**症状：** `api.github.com` GET 正常，POST 超时

**学到的教训：**
1. 网络问题 ≠ 全断 — 一定要分别测试 GET/POST/TCP
2. 不同端口可能走不同路由 — 443/80 可能通，POST 端点被单独屏蔽
3. 设备码流程（OAuth Device Flow）需要 POST → 不适用时换 token 直接写入
4. `gh auth --with-token` 在 Windows PowerShell 里需要注意编码问题

**快速检测脚本：**
```powershell
$tests = @(
    @{Name="GitHub.com";    Url="https://github.com"; Method="GET"},
    @{Name="api.github.com"; Url="https://api.github.com/zen"; Method="GET"},
    @{Name="GitHub POST";   Url="https://github.com/login/device/code"; Method="POST"}
)
foreach ($t in $tests) {
    try {
        $sw = [Diagnostics.Stopwatch]::StartNew()
        if ($t.Method -eq "GET") {
            $r = Invoke-WebRequest -Uri $t.Url -TimeoutSec 8 -UseBasicParsing -MaximumRedirection 0 -ErrorAction SilentlyContinue
        } else {
            $r = Invoke-WebRequest -Uri $t.Url -Method POST -TimeoutSec 8 -UseBasicParsing -MaximumRedirection 0 -ErrorAction SilentlyContinue
        }
        $sw.Stop()
        Write-Host "$($t.Name): OK ($($sw.ElapsedMilliseconds)ms) - $($r.StatusCode)"
    } catch {
        Write-Host "$($t.Name): FAIL - $($_.Exception.Message)"
    }
}
```
